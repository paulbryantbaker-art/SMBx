01

Philosophy

Three principles shape every decision. When a question comes up that these don't answer, err toward restraint.

Chrome is greyscale. Content is where color lives.

A deal scoring dashboard carries meaningful signal: green, amber, red. A financial chart has real data to visualize. A document preview has its own colors and identity. That's where color belongs — in the substance of what the user is looking at. UI chrome — tab bars, buttons, nav, section headings — stays monochrome so the content can speak. This is the Bloomberg Terminal principle, softened for consumer use.

Glass floats. Content is solid.

Apple's Liquid Glass language is used for chrome that floats over content: the tab bar, the Yulia mini bar, the nav pill, icon buttons, sheets, and overlays. Content surfaces — cards, document bodies, hero sections — stay solid white. This keeps readability uncompromised while the navigation layer feels alive and present without fighting for attention.

Typography carries hierarchy. Not color.

Big numbers are big and black, not colored. Section headers are bold and black, not colored. Active states are darker, not colored. When something needs to feel important, it gets more weight, more size, or more contrast — never a tint. The one exception is the "winning" cell pattern (offer comparison, Blind Equity) where black fill + white text signals emphasis without introducing hue.

The one rule

If you're about to add a color to solve a hierarchy problem, change the typography or contrast instead. If you genuinely need color, it belongs in the content, not the chrome.

05

Glass surfaces

The signature element. Three ingredients, always together: translucent white background, heavy backdrop blur with saturation, and a specular inset top highlight. Miss any one and it stops reading as glass.

The recipe

/* Standard glass — tab bar, mini bar, nav pill */
background: rgba(255, 255, 255, 0.72);
backdrop-filter: blur(32px) saturate(1.8);
-webkit-backdrop-filter: blur(32px) saturate(1.8);
border: 0.5px solid rgba(0, 0, 0, 0.08);
box-shadow:
  /* the specular highlight — don't skip this */
  inset 0 0.5px 0 rgba(255, 255, 255, 0.9),
  /* outer shadow only on floating chrome */
  0 6px 24px rgba(0, 0, 0, 0.08),
  0 1px 3px rgba(0, 0, 0, 0.05);

The specular highlight

The inset 0 0.5px 0 rgba(255,255,255,0.9) is what makes translucent surfaces read as glass instead of flat overlays. Without it, the surface looks like a tinted pane. With it, the top edge catches light like real glass would. Never skip this line.

When to use glass

Glass is for chrome that floats over content: the tab bar, the Yulia mini bar, the public nav pill, floating CTAs, sheets and popovers, icon buttons inside a topbar over rich content.

When not to use glass

Content surfaces stay solid. Document bodies, deal cards, stat cards, input fields — all solid white with a hairline border and subtle inset top highlight. Glass everywhere becomes noise; solid cards with glass chrome creates the depth.

Rules

Always lowercase except for the X when used alone as a mark. Letter-spacing -0.025em. Weight 800. No colored letter — the brand is the word, not a visual device. The logo inherits its color from the containing surface: black on light, white on dark.

09

Motion

Minimal. Quick. Confident. Animations exist to acknowledge an action, not to entertain.

Timing

/* Instant feedback — chips, buttons, hover states */
transition: all 150ms cubic-bezier(0.4, 0, 0.2, 1);

/* Structural — tab switches, sheet open/close, mini bar expand */
transition: all 300ms cubic-bezier(0.32, 0.72, 0, 1);

/* Page morph — public → app */
transition: opacity 400ms ease, transform 400ms cubic-bezier(0.32, 0.72, 0, 1);

Signature motions

Glass shell frost-in · the tab bar and mini bar appear by fading up from below with a brief backdrop-filter growth from blur(0px) to blur(32px). 400ms.

Yulia state change · full chat → mini bar uses layoutId on the avatar so it morphs between sizes without a fade. 300ms.

Chat input send · field content lifts up as a user bubble, the input clears instantly. Yulia's response streams in character by character, not word by word.

Reduced motion · respect prefers-reduced-motion. Keep opacity fades; kill transforms and blurs.

10

Do / don't

The quickest way to learn the system is by contrast.

Do

Dark text on white card for the baseline number

The emphasis comes from typography weight (Sora 800) and size (30px+). It reads authoritative without borrowing color.

Don't

Color the baseline number for emphasis

Terra, blue, any accent hue on the hero number turns content into chrome. Stop. The number is content; it gets its power from being the largest thing on the screen.

Do

Solid white cards with hairline borders

Content surfaces need to be readable. Solid white with rgba(0,0,0,0.08) border and inset top highlight. Always.

Don't

Apply glass to every surface

Glass everywhere becomes visual mud. Glass is for floating chrome. Content is solid. The contrast between the two creates the depth.

Do

Plain text logo

smbx.ai in Sora 800, inheriting surrounding color. The wordmark is the logo.

Don't

Color one letter

No terra X. No accent on the X. No gradient on the word. If the containing surface is light, logo is black; if dark, white. Full stop.

Do

Status dot + dark text

A 7px colored dot carries the signal. The pill itself is white with a hairline border and dark text. Minimal, financial-terminal restraint.

Don't

Full green/red/amber pill backgrounds

Fills introduce color into the chrome. Scale up four status types with full color fills and the UI becomes a traffic light. The dot is enough.

Do

Specular highlight on glass

The inset 0 0.5px 0 rgba(255,255,255,0.9) makes translucent surfaces read as glass. Non-negotiable.

Don't

Skip the inner highlight

Without the inset white line on the top edge, your "glass" is just a tinted overlay. The highlight is what sells it.

Do

Typography carries hierarchy

Big things are big and bold. Small things are small and muted. Active things are darker. Use weight and size to signal importance.

Don't

Color things to make them "pop"

If you need something to pop, make it bigger, heavier, or darker. Color is a content concern, not a chrome concern.
